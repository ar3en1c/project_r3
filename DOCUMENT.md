# راهنمای جامع فایل‌های `index.html` و `series.scss`

این سند راهنمایی کامل به زبان فارسی برای درک ساختار، اجزا و نحوه طراحی دو فایل زیر در پروژه است:
1. قالب HTML جزئیات سریال: [index.html](file:///G:/Html/project_r3/series/templates/series/index.html)
2. استایل‌های SCSS صفحه سریال: [series.scss](file:///G:/Html/project_r3/static/scss/series.scss)

طراحی این بخش از وب‌سایت دارای یک تم علمی-تخیلی/ترمینال (Sci-Fi/Cyberpunk Terminal) با ترکیب فونت‌های تک‌فاصله (Monospace)، متغیرهای سیستمی و رنگ‌های هماهنگ است.

---

## بخش اول: بررسی فایل قالب HTML (`index.html`)

این فایل قالب جنگو (Django Template) برای رندر کردن صفحه اطلاعات و جزئیات یک سریال (در اینجا Breaking Bad) است.

### ۱. ارث‌بری و بارگذاری ابزارها
- **خط ۱ (`{% extends "base.html" %}`)**: این قالب ساختار کلی خود را از فایل قالب پایه پروژه یعنی `base.html` به ارث می‌برد.
- **خط ۲ (`{% load static %}`)**: ابزار بارگذاری فایل‌های استاتیک را فعال می‌کند تا بتوان تصاویری مثل پوسترها و استایل‌ها را فراخوانی کرد.
- **خط ۳ (`{% load render_partial %}`)**: امکان رندر کردن بخش‌های کوچک‌تر و مجزا (مانند هدر و فوتر) را فراهم می‌کند.

### ۲. استایل‌های اختصاصی صفحه
- **بلاک `head` (خط ۷ تا ۹)**: فایل استایل کامپایل شده `static/css/series.css` را در بخش `<head>` صفحه قرار می‌دهد.

### ۳. بخش بدنه اصلی صفحه (`{% block body %}`)
ساختار بدنه شامل سه بخش عمده زیر است:
- **هدر سیستم (`{% render_partial "series:header" %}`)**: هدر کلی سایت را در بالای صفحه رندر می‌کند.
- **محتوای اصلی (`<main class="main-container">`)**: کانتینر اصلی که تمام جزئیات سریال و سایدبار را در بر می‌گیرد.
- **فوتر سیستم (`{% render_partial "series:footer" %}`)**: فوتر سایت را در انتهای صفحه رندر می‌کند.

---

### ۴. ساختار درونی محتوای اصلی (`.main-container`)

محتوای اصلی به دو بخش `.content-area` (سمت چپ/وسط) و `.sidebar` (سمت راست) تقسیم می‌شود.

#### الف) بخش محتوا (`.content-area`)

این بخش شامل دو قسمت بزرگ است:

##### ۱. بخش قهرمان یا هیرو (`.hero-section`)
این بخش در بالای صفحه قرار دارد و اطلاعات شاخص سریال را نشان می‌دهد:
* **تصویر زمینه (`.hero-image`)**: پوستر اصلی سریال Breaking Bad.
* **مشخصات فنی (`.tech-specs`)**: نمایش متونی مانند کیفیت پخش و نرخ فریم به همراه سیگنال زنده سیستم (`LIVE_SIGNAL`).
* **عنوان سریال (`.series-title`)**: عنوان بزرگ سریال.
* **ژانرها (`.genre-tags`)**: تگ‌های ژانر سریال (Western, Thriller, Crime, Drama).
* **کنترل وضعیت تماشا (`.tracking-section`)**:
  * دکمه‌های انتخاب وضعیت (COMPLETED، WATCHING که به صورت پیش‌فرض فعال است، DROPPED، PLAN_TO_WATCH).
  * پیشرفت تماشا (`80 / 45`) به همراه برچسب همگام‌سازی قسمت‌ها.
  * دکمه ثبت پیشرفت (`LOG_INCREMENT +`).
* **آمارهای سیستمی در پایین هیرو (`.bottom-stats`)**: پهنای باند ویدیو، میزان تاخیر شبکه و موقعیت سرور.

##### ۲. بخش کارت‌های اطلاعات (`.info-section`)
این بخش به صورت دو ستونه (یک ستون اطلاعاتی در سمت چپ و یک ستون محتوایی در سمت راست) طراحی شده است:

* **ستون سمت چپ (`.left-column`)**:
  * **کارت امتیاز (`rating-card`)**: نمایش امتیاز کاربر به صورت ستاره‌ای (امتیاز ۹.۵ از ۱۰).
  * **کارت وضعیت پخش (`status-card`)**: نمایش وضعیت پخش سریال (Ended // به اتمام رسیده) به همراه چرخه‌های زمانی و سال انتشار (2008).
  * **کارت متادیتای سیستم (`metadata-card`)**: اطلاعات مرجع مانند لینک IMDB، زبان اصلی سریال، منطقه تولید و تاریخ بارگذاری روی سرور.
  * **کارت بازیگران فرعی (`supporting-card`)**: بازیگران مکمل مهم سریال (مانند Gustavo Fring و Mike Ehrmantraut) به همراه تصاویر کوچک آواتار و نام بازیگران.

* **ستون سمت راست (`.right-column`)**:
  * **کارت خلاصه داستان (`overview-card`)**: توضیحات کلی در مورد داستان سریال Breaking Bad.
  * **کارت شخصیت‌های کلیدی (`characters-section`)**: نمایش شبکه بازیگران اصلی سریال (Walter White, Jesse Pinkman, Saul Goodman) به همراه عکس و نام کاراکترها در یک گرید جذاب.

#### ب) سایدبار فرمان سنگی (`.sidebar`)

## بخش دوم: بررسی فایل جاوااسکریپت (بلاک `script`)

در انتهای فایل `index.html` یک بلاک `<script>` وجود دارد که مسئول مدیریت **سیستم امتیازدهی تعاملی** است.

### ۱. دریافت المان‌های DOM (`get_rating_elements`)
```javascript
function get_rating_elements() {
  return {
    scoreButton: document.querySelector(".rating-value"),
    popover: document.getElementById("rating-popover"),
    slider: document.getElementById("rating-slider"),
    popoverValue: document.getElementById("rating-popover-value"),
  };
}
```
این تابع کمکی، چهار المان اصلی سیستم امتیازدهی را از DOM استخراج می‌کند و به صورت یک آبجکت برمی‌گرداند.

### ۲. تنظیم امتیاز (`set_rating`)
```javascript
function set_rating(rate = null) {
  const { scoreButton, slider, popoverValue } = get_rating_elements();
  const min = Number(slider.min);   // حداقل: 0
  const max = Number(slider.max);   // حداکثر: 10
  const nextRate = Math.min(max, Math.max(min, Number(rate)));
  // ...
}
```
این تابع مقدار امتیاز را دریافت و:
- مقدار اسلایدر را به‌روز می‌کند
- نوار پیشرفت CSS را تنظیم می‌کند (`--rating-progress`)
- متن دکمه امتیاز و مقدار نمایشی در پاپ‌آپ را تغییر می‌دهد
- از `Math.min` و `Math.max` برای محدود کردن مقدار بین 0 و 10 استفاده می‌کند

### ۳. تغییر امتیاز با دکمه‌ها (`change_rating`)
```javascript
function change_rating(step) {
  const { slider } = get_rating_elements();
  set_rating(Number(slider.value) + step);
}
```
با کلیک روی دکمه‌های `+` و `-` مقدار اسلایدر به اندازه `step` (مثبت یا منفی) تغییر می‌کند.

### ۴. نمایش/مخفی کردن پاپ‌آپ (`toggle_rating_popover`)
```javascript
function toggle_rating_popover() {
  const { popover, slider } = get_rating_elements();
  popover.hidden = !popover.hidden;
  if (!popover.hidden) {
    set_rating(slider.value);
    slider.focus();
  }
}
```
با کلیک روی دکمه امتیاز:
- پاپ‌آپ نمایش/مخفی می‌شود
- در صورت نمایش، اسلایدر فوکوس می‌گیرد

### ۵. بستن پاپ‌آپ (`close_rating_popover`)
```javascript
function close_rating_popover() {
  const { popover } = get_rating_elements();
  popover.hidden = true;
}
```
پاپ‌آپ را مخفی می‌کند. این تابع هم با کلیک روی پس‌زمینه و هم با کلید Escape فراخوانی می‌شود.

### ۶. رویداد کیبورد (Escape)
```javascript
document.addEventListener("keydown", function (event) {
  if (event.key === "Escape") {
    close_rating_popover();
  }
});
```
با فشردن کلید Escape از هر نقطه‌ای در صفحه، پاپ‌آپ امتیازدهی بسته می‌شود.

## بخش سوم: بررسی فایل استایل SCSS (`series.scss`)

این فایل استایل‌های مورد نیاز برای رندر صفحات با ساختار بالا را مدیریت می‌کند و از قابلیت‌های پیش‌پردازنده Sass (مانند Nesting و Variables) استفاده می‌برد.

### ۱. متغیرها و تنظیمات پایه
* **خط ۱ (`@use 'var' as *;`)**: تمام متغیرهای رنگی و اندازه‌دهی پروژه را از فایل `_var.scss` وارد می‌کند (رنگ اصلی سیستم `$primary`، پس‌زمینه `$background`، و خطوط مرزی `$border`).
* **تنظیم بدنه (`body`)**: فونت کل وب‌سایت روی حالت مونو اسپیس (`'Courier New', monospace`) تنظیم شده است تا حس ترمینال‌های هکری یا قدیمی را بازسازی کند.

### ۲. استایل‌های ناوبری و هدر
* **`.header`**: با چیدمان Flexbox لوگو، لینک‌ها و فیلد جستجو را کنار هم می‌چیند. لینک فعال دارای رنگ متمایز اصلی است.
* **`.sidebar`**: عرض ثابت `200px` داشته و در سمت راست صفحه با ویژگی `position: fixed` چسبانده شده است.

### ۳. استایل‌های بخش قهرمان (`.hero-section`)
* **`.hero-overlay`**: یک لایه گرادیان عمودی تیره روی تصویر اصلی ایجاد می‌کند تا متون سفید به خوبی خوانده شوند.
* **`.status-btn`**: دکمه‌های کنترلی وضعیت تماشا که در صورت فعال بودن (`.active`) رنگ پس‌زمینه اصلی سیستم و نوشته تیره به خود می‌گیرند.

### ۴. طراحی دو ستونه و کارت‌ها
* **`.info-section`**: با استفاده از CSS Grid محتوا را به دو ستون ۲۸۰ پیکسلی (برای کارت‌ها) و بخش کشسان باقی‌مانده (برای خلاصه داستان و بازیگران) تقسیم می‌کند.
* **`.character-card`**: کارت بازیگران اصلی که عکس آن‌ها دارای افکت نیمه‌سیاه‌وسفید (`filter: grayscale(30%)`) است و در زمان هاور، بوردر دور آن روشن‌تر و به رنگ اصلی تغییر می‌کند.

### ۵. ریسپانسیو و واکنش‌گرایی صفحه (`@media`)
این فایل استایل به صورت کاملا واکنش‌گرا نوشته شده است:
* **در عرض‌های کمتر از 1200px**: عرض سایدبار از ۲۰۰ به ۱۶۰ پیکسل کاهش می‌یابد تا فضای محتوای میانی بزرگتر بماند.
* **در عرض‌های کمتر از 992px (موبایل‌ها و تبلت‌ها)**:
  * سایدبار سمت راست کلاً پنهان می‌شود (`display: none`).
  * فاصله سمت راست محتوای اصلی به صفر تغییر می‌کند تا صفحه تمام‌عرض شود.
  * ساختار دو ستونه کارت‌ها و متن داستان به صورت تک‌ستونه در می‌آید تا محتوا به راحتی زیر هم نمایش داده شود.
  * گرید بازیگران از ۳ ستون به ۲ ستون تبدیل می‌شود.
* **در عرض‌های کمتر از 576px (موبایل‌های کوچک)**:
  * ارتفاع هیرو سکشن به ۴۰۰ پیکسل تقلیل یافته و فونت عنوان اصلی سریال کوچکتر می‌شود.
  * کارت بازیگران اصلی به صورت تک‌ستونه چیده می‌شود.
